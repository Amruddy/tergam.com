# dds-tracker

Сжатый контекст по текущему состоянию сайта. Это snapshot по коду на 2026-04-19.

## Как использовать
- Сначала читать `Context.md`.
- Для фич смотреть `Features.md`.
- Для хранилища и sync смотреть `Data Flow.md`.
- Для проблем и рисков смотреть `Known Issues.md`.
- Для ближайших задач смотреть `TODO.md`.
- Для истории рабочих решений смотреть `Decisions.md`.
- Для короткого журнала изменений по дням смотреть `Daily.md`.
- Это заметки для экономии токенов: короткие, без дублирования JSX.

## Snapshot
- Стек: Next.js 14, React 18, TypeScript, Tailwind, Zustand, Recharts, Framer Motion, Supabase.
- Основной сценарий: учёт личных финансов с локальным store и облачной синхронизацией при наличии Supabase env.
- Рабочее дерево на момент генерации: чистое.

## Главные файлы
- `app/` — роуты.
- `components/` — экраны и UI.
- `store/useTransactionStore.ts` — единый client store.
- `lib/db.ts` — Supabase CRUD и загрузка.
- `types/index.ts` — доменные типы.

## Рабочий набор
- `TODO.md` — что делать дальше.
- `Decisions.md` — почему были приняты конкретные решения.
- `Daily.md` — краткий дневник изменений.
- `Task Template.md` — заготовка под новую задачу.
