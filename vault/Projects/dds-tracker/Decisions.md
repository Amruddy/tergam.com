# Decisions

## 2026-04-19 — Компактный vault вместо длинных сводок
- Решение: хранить короткие заметки по проекту рядом с кодом в отдельном `vault`.
- Почему: это дешевле по токенам, чем каждый раз перечитывать и пересобирать структуру приложения из исходников.
- Последствия: любые значимые изменения в архитектуре, data flow или UX стоит отражать в vault сразу после правок.

## 2026-04-19 — Структура заметок разделена по назначению
- Решение: разделить контекст на `Context`, `Features`, `Data Flow`, `Known Issues`, `TODO`, `Daily`, `Decisions`.
- Почему: так проще открывать только нужный слой контекста под конкретную задачу.
- Последствия: не складывать всё в один большой файл, иначе экономия токенов пропадает.

## 2026-04-19 — Vault хранится рядом с проектом, а не внутри репозитория
- Решение: использовать `C:\Users\Mi\Desktop\Проекты\vault` как отдельную папку.
- Почему: не смешивать git-историю приложения и рабочие заметки.
- Последствия: заметки доступны локально, но не попадают в репозиторий автоматически.
